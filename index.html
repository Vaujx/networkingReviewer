Introduction to Communications and Networks

Communication is a process that involves sending and receiving messages through verbal and non-verbal methods. Data communication examines how data, in the form of energy, travels across some medium from a source to a destination.

A Simplified Communications Model

  • Source: Generates data to be transmitted
  • Transmitter: Converts data into transmittable signals
  • Transmission System: Carries data
  • Receiver: Converts received signal into data
  • Destination: Takes incoming data

Brief History of Communications

  • 1837: Samuel Morse invented the telegraph with Morse code
  • 1876: Alexander Graham Bell invented the telephone
  • 1945: ENIAC was invented as the first electronic computer
  • 1980s: Personal Computers became a new way of computing
  • 1990s: Emergence of the Internet and World Wide Web applications

Communication Devices

A communication device is hardware capable of transmitting analog or digital signals over telephone, communication wire, or wireless.

  • Bluetooth Devices
  • Infrared Devices
  • Modem (Modulator-Demodulator)
  • Network Card or NIC (Network Interface Card)
  • Smart Phone
  • Wi-Fi Devices

What are transmitted via Communication Media?

  • Voice: Encoded signals (telephone, FAX, video conferencing, cellular phones)
  • Video: Encoded images (television, VCR, DVD)
  • Data: Character streams (LAN, WAN, Internet)

OSI Model (Open System Interconnection)

The OSI Model was developed by the International Organization for Standardization (ISO 7498:1984). It consists of seven layers, each with a different but specific processing function. The OSI Model describes the flow of data in a network, from the lowest layer up to the layer containing the user's applications.

Layer 7: Application Layer

Responsible for providing networking services to the user. Also known as the Desktop Layer. Identification of services is done using Port Numbers.

Examples of Networking Services:

  • HTTP (Port 80): Hypertext Transfer Protocol - allows users to communicate data on the World Wide Web
  • FTP (Port 21): File Transfer Protocol - used for transferring files from a server to a client
  • SMTP (Port 25): Simple Mail Transfer Protocol - used for sending email messages
  • TELNET (Port 23): Teletype Network - provides virtual access to a computer
  • TFTP (Port 69): Trivial File Transfer Protocol - used to transfer files over networks

Layer 6: Presentation Layer

Responsible for converting data into standard format. Examples: ASCII, EBCDIC, JPEG, MPEG, BMP, MIDI, WAV, MP3.

Tasks:

  • Encoding-Decoding: Converting characters into specialized format for efficient transmission
  • Encryption-Decryption: Encoding plaintext into ciphertext using algorithms and keys
  • Compression-Decompression: Reducing file size for storage or transmission

Layer 5: Session Layer

Responsible for establishing, maintaining, and terminating sessions. Session ID works at this layer.

Examples:

  • RPC (Remote Procedure Call): Protocol for high-level communications in operating systems
  • SQL (Structured Query Language): Used to communicate with databases
  • NFS (Network File System): Protocol for distributed file sharing

Layer 4: Transport Layer

Responsible for end-to-end connectivity. Also known as the heart of OSI Layers.

Tasks:

  • Identifying Service: Using protocols like TCP (connection-oriented) and UDP (connectionless)
  • Multiplexing & De-multiplexing: Transmitting data from different applications simultaneously
  • Segmentation: Dividing messages into segments
  • Sequencing & Reassembling: Ordering segments and reconstructing the original message
  • Error Correction: Detecting and fixing transmission errors
  • Flow Control: Managing data transmission rates between nodes

Layer 3: Network Layer

Responsible for providing the best path for data to reach the destination. Logical Addressing works on this layer. Router is a Network Layer device.

Divided into two parts:

  • Routed Protocols: IP, IPX, Apple Talk
  • Routing Protocols: RIP, IGRP, OSPF, EIGRP

Layer 2: Data Link Layer

Divided into two sub-layers:

  • LLC (Logical Link Control): Talks about WAN Protocols (PPP, HDLC, Frame-Relay)
  • MAC (Media Access Control): Talks about Physical Address (48-bit address, 12-digit Hexadecimal Number)

Also responsible for Error Detection (CRC Check). Devices: Switch, Bridge, and NIC.

Layer 1: Physical Layer

Responsible for electrical, mechanical, and procedural checks. Data is converted into Binary 0's and 1's.

Data forms:

  • Electrical Pulses: Coaxial or Twisted Pair Cable
  • Light: Fiber Optic Cable

Devices: Hubs, Repeaters, Cables, Modems, and others.

IP Addressing

IP Addressing is Logical Addressing that works on Network Layer (Layer 3). There are two versions of addressing schemes:

  • IP Version 4: 32-bit addressing
  • IP Version 6: 128-bit addressing

IP Version 4 (IPv4)

32 bits are divided into 4 octets known as Dotted Decimal Notation. Each octet represents 8 bits (values from 0 to 255).

IP Version 6 (IPv6)

128-bit address is divided along 16-bit boundaries, and each 16-bit block is converted to a 4-digit hexadecimal number and separated by colons (Colon-Hex Notation).

Example: ABCD:34AC:1234:FEAB:AB89:4567:3245:7896

Private IP Addresses

Addresses reserved for LANs that are not connected to the Internet:

  • Class A: 10.0.0.0 to 10.255.255.255
  • Class B: 172.16.0.0 to 172.31.255.255
  • Class C: 192.168.0.0 to 192.168.255.255

Subnet Mask

Subnet Mask differentiates Network portion and Host portion. It is represented with all 1's in the network portion and all 0's in the host portion.

  • Class A: 255.0.0.0
  • Class B: 255.255.0.0
  • Class C: 255.255.255.0

Subnetting

Subnetting is dividing a single network into multiple networks by converting host bits to network bits. It can be done based on:

  1. Requirement of Networks
  2. Requirement of Hosts
  3. CISCO / Notation

Network Types and Topologies

Types of Networks

  • Local Area Network (LAN): Connects a relatively small number of machines in a close geographical area
  • Wide Area Network (WAN): Connects LANs over potentially large geographic distances
  • Metropolitan Area Network (MAN): Communication infrastructures developed in and around large cities

Network Topologies

  • Bus Topology: Nodes are connected to a single communication line that carries messages in both directions
  • Star Topology: Centers around one node to which all others are connected and through which all messages are sent
  • Ring Topology: Connects all nodes in a closed loop in which messages travel in one direction
  • Fully Connected Topology: A direct path exists between any two nodes
  • Hybrid Topology: Uses two or more different network topologies

Communication Software

Communication software is an application or program designed to pass information from one system to another. It includes systems and applications that enable the exchange of text, audio, video messages, and files via the cloud or local networks.

Tools such as email, chat, messaging apps, FTP systems, and VoIP platforms help users communicate across devices in real-time.

Protocols

A protocol is a set of rules or procedures for transmitting data between electronic devices. There are three main types:

  • Network Management Protocols
  • Network Communication Protocols
  • Network Security Protocols

Communication protocols include basic data communication tools like TCP/IP and HTTP.
